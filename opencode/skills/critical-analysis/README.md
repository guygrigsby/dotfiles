# Critical Analysis Skill

Rigorous adversarial analysis of technical arguments, architectural decisions, and engineering proposals.

## Purpose

This skill helps identify:
- Logical fallacies in technical arguments
- Unexamined assumptions
- Edge cases and failure modes
- Hidden trade-offs
- Conceptual weaknesses

## When to Use

Invoke this skill when you need to:
- Challenge architectural decisions
- Stress-test API designs
- Critique technical proposals
- Find holes in requirements
- Play devil's advocate

## Usage

Trigger phrases:
- "poke holes in this"
- "what's wrong with"
- "challenge this idea"
- "play devil's advocate"
- "critique this"
- "what am I missing"
- "break it"

## Analysis Modes

1. **Gentle Critique** (default) - Diplomatic, collaborative
2. **Rigorous Challenge** - Aggressive, no sacred cows
3. **Security Mindset** - Adversarial attacker perspective
4. **Operational Perspective** - "Who gets paged at 3am?"

## Framework

The skill uses a 6-phase analysis framework:
1. Clarification (Steel Man)
2. Structural Analysis (Logical Fallacies)
3. Assumptions & Constraints
4. Edge Cases & Failure Modes
5. Trade-off Analysis
6. Alternative Perspectives

## Integration

Works well with:
- `domain-driven-design` - Critique DDD designs
- `oltp-schema-design` - Challenge schema choices
- `olap-schema-design` - Question dimensional models
- `requirements-clarity` - Validate requirements
- `gepetto` - Review implementation plans
